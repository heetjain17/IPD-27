- store place data

- search nearby places

- latitude/longitude handling
- radius queries
- distance calculation
- sorting

## 5. Saved Places Module

Derived from **User Interaction Layer (partial)**

# Data flow

## Flow 1: Nearby Places

```text
User → API (/places/nearby)
→ validate lat/lng
→ query DB (geo)
→ return sorted places
```

---

## Flow 2: Save Place

```text
User → API (/places/save)
→ auth middleware
→ check place exists
→ insert into saved_places
→ return success
```

---

## Flow 3: Get Saved

```text
User → API (/places/saved)
→ auth
→ fetch saved_places
→ join places
→ return list
```

---

## Flow 4: Login

```text
User → API (/auth/login)
→ validate credentials
→ generate token
→ return token
```

---

# DB constraints

## Location constraints

- lat/lng stored as numeric
- need indexing for radius queries
- PostGIS enabled

## User constraints

- email unique
- password hashed

## Saved places

- no duplicates (user_id + place_id unique)
